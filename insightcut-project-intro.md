## 1. 项目愿景 (Project Vision)

**让每一个观点，都能成为一条高质量解说视频。**

InsightCut 是一款面向知识创作者与内容生产者的 AI 解说视频生成工具，打通「观点表达 → 脚本生成 → 视频成片」的完整链路。

知识类短视频需求旺盛，但制作门槛高：写脚本、拆分镜、找素材、配音、字幕、剪辑，每一步都需要专业工具和经验。InsightCut 解决的核心问题是：**有观点想表达，但没有完整视频生产能力**。

**InsightCut 的解法**：从一个观点、一段文案或一个选题出发，自动完成脚本改写、分镜拆解、AI 图片生成、TTS 配音、字幕生成、MP4 成片和剪映草稿导出。

### ✅ 已完成能力

*   **短视频节奏分段**：按解说节奏自动切分段落，每段生成画面描述和配音文本。
*   **高质量 AI 图片生成**：基于视频主题、段落文本和画面风格生成分镜提示词，调用生图模型生成高质量图片。

*   **资产库管理**：保存所有生成的图片、音频、字幕和草稿资产，支持查看、下载和复用。
*   **分镜编辑能力**：支持修改字幕文案、重新生成图片、上传本地图片替换、重新生成配音。

## 3. AI Agent 架构设计

InsightCut 的核心竞争力是将多个 AI 能力串联成一条稳定的视频生产流水线：脚本生成 Agent、分镜设计 Agent、素材生成 Agent。

### 📝 Agent 1：脚本生成 Agent

**职责**：将原始观点或文案整理成适合解说的视频脚本。

```text
用户输入主题或文案
  ↓
AI 分析内容结构与核心观点
  ↓
改写为解说风格脚本
  ↓
提取视频主题和总结
  ↓
保存到项目数据库
```

**设计亮点**：
*   **语气优化**：将书面语转化为口语化表达，更适合视频解说。
*   **节奏控制**：自动调整句子长度和段落结构，符合短视频节奏。
*   **观点提炼**：提取核心观点作为视频主题，便于后续画面生成。

### 🎬 Agent 2：分镜设计 Agent

**职责**：将脚本拆解为可视化的分镜序列。

```text
START
  ↓
analyze_script（分析脚本结构）
  ↓
split_segments（按节奏切分段落）
  ↓
generate_visual_prompts（生成画面描述）
  ↓
estimate_duration（估算每段时长）
  ↓
save_segments（保存分镜数据）
  ↓
END
```

**设计亮点**：
*   **语义理解**：不是简单按字数切分，而是识别语义完整性和节奏点。
*   **画面匹配**：为每段生成贴合内容的画面描述，而非通用占位图。
*   **时长控制**：根据配音文本长度估算每段时长，确保节奏合理。

### 🎨 Agent 3：素材生成 Agent

**职责**：为每个分镜生成高质量图片和配音。

```text
START
  ↓
generate_image_prompts（生成图片提示词）
  ↓
call_image_model（调用生图模型）
  ↓
generate_voiceover（调用 TTS 生成配音）
  ↓
generate_subtitles（生成字幕）
  ↓
save_assets（保存素材到资产库）
  ↓
END
```

**设计亮点**：
*   **提示词优化**：基于视频主题、段落文本和画面风格生成高质量提示词。
*   **风格一致性**：确保所有分镜图片风格统一，不会出现画风跳跃。
*   **配音自然度**：使用高质量 TTS 模型，确保配音流畅自然。
*   **资产沉淀**：所有生成的素材都保存到资产库，支持后续复用。

## 4. 典型用户工作流

1. **创作准备阶段**：用户在「新建项目」输入视频主题或粘贴原始文案，点击「生成脚本」。
2. **脚本确认阶段**：AI 生成解说脚本和视频总结，用户可以修改调整。
3. **分镜生成阶段**：点击「生成分镜」，AI 自动拆解脚本、生成画面描述、生成图片和配音。
4. **编辑优化阶段**：在分镜编辑页面，用户可以修改字幕、重新生成图片、上传本地图片、重新生成配音。
5. **导出发布阶段**：点击「导出 MP4」直接生成成片，或点击「导出剪映草稿」继续精修。

## 5. 核心特点

### 🎯 高质量 AI 图片

InsightCut 把画面生成作为核心能力之一，而不是附属占位图。系统会基于视频主题、段落文本、内容总结和画面风格生成分镜提示词，尽量让每张图都服务于当前解说内容。

默认目标是：图片清晰、风格统一、语义贴合、能直接进入视频，而不是后期再大量筛图。

### ⚡ 默认直出效果好

InsightCut 默认会处理字幕、画面比例、基础镜头运动、音频对齐和 MP4 导出。用户不需要先理解复杂时间线，也不需要手动拖拽素材，就能得到一条可以预览、评估甚至直接发布的成片。

### ✏️ 支持详细二次修改

一键生成之后，用户仍然可以继续细调：

*   修改每个分镜的字幕文案
*   修改图片提示词并重新生成图片
*   上传本地图片替换某个分镜
*   重新生成某段配音
*   在历史素材中选择旧图替换当前画面
*   进入导出中心重新生成 MP4 或剪映草稿

这让 InsightCut 不是"一次性生成器"，而是可反复打磨的视频工作台。

### 💾 资产沉淀

每次生成的视频都会留下结构化资产：图片、音频、字幕、分镜文本、提示词、导出文件和任务记录。资产库可以记录成功和失败的素材状态，方便后续继续修复、复用和追踪。

这也是产品和普通一键成片工具的重要区别：生成过程不是一次性消耗，而是在不断积累可复用资产。

### 🎬 MP4 和剪映草稿双出口

InsightCut 支持两种结果出口：

*   **直出 MP4**：适合快速预览、内部评审、直接发布或验证选题。
*   **导出剪映/CapCut 草稿**：适合进入专业剪辑流程，继续精修节奏、贴纸、转场、音乐和包装。

所以它既能服务轻量创作，也能接入更完整的后期流程。

## 6. 技术架构

| 层级 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **Frontend** | Vue 3 + TypeScript + Vite | 现代化前端工具链 |
| **UI** | Element Plus | 快速构建响应式界面 |
| **State** | Composition API | 管理项目、分镜、素材等全局状态 |
| **Backend** | FastAPI + Python | 高性能异步 API 服务 |
| **Database** | SQLite / PostgreSQL | 本地开发使用 SQLite，生产环境支持 PostgreSQL |
| **Agent Orchestration** | Anthropic Claude API | 脚本生成、分镜设计、提示词优化 |
| **Image Generation** | 兼容 OpenAI 格式的生图 API | 支持多种生图模型 |
| **TTS** | 字节跳动 TTS API | 高质量中文配音 |
| **Video Export** | FFmpeg + pyjianyingdraft | MP4 导出和剪映草稿生成 |
| **Storage** | 本地文件 / 阿里云 OSS / 腾讯云 COS | 支持本地和云端素材存储 |
